# 2026-03-14 工作记录

## 服务器安全加固 🛡️
- 对百度云服务器 106.12.13.23 进行安全审计
- 发现严重隐患：防火墙关闭、SSH允许密码登录、fail2ban未安装
- 正在被暴力破解：发现多个IP在尝试SSH登录攻击

- 创建了「SSH加固技能」记录经验教训，以后不会再犯同样的错误

## 防火墙配置 🔥
- 开启 firewalld 防火墙
- 开放端口：80(HTTP)、443(HTTPS)、22(SSH)、781(百度云监控)
- 服务器现在有了基本防护

## SSH密钥配置 🔑
- 生成 ED25519 密钥对（更安全更快）
- 配置 authorized_keys
- 测试密钥登录成功
- **私钥已保存**：/root/.openclaw/workspace/ssh_key_106.12.13.23

## fail2ban 防暴力破解 🚫
- 安装并配置 fail2ban
- 设置规则：10分钟内失败5次封禁1小时
- 自动封禁暴力破解IP

## OpenClaw 定时重启 ⏰
- 设置每天凌晨6:00自动重启 OpenClaw Gateway
- 防止服务挂掉失联
- 日志记录到 /var/log/openclaw-restart.log

## 备用API配置 🤖
- 配置硅基流动作为备用模型
- 免费模型：Qwen2.5-7B-Instruct、GLM-4-9B-Chat
- 防止 GLM-5 token 用完后失联
- 主用：GLM-5，备用：硅基流动

## 创建技能 📚
- 新建 SSH安全加固技能 (skills/ssh-hardening/SKILL.md)
- 核心教训：先测试密钥登录，再禁用密码
- 使用 PermitRootLogin prohibit-password 而不是 no
- 改配置前必须用 sshd -t 测试语法
